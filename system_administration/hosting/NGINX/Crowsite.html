creating a reverse proxy

now from what i understand, a proxy is basically just a routing layer in front of the server, to direct to different ports and other servers

Nginx supports this and then some so here is what to do

  1. have your server bloc be in a random port rather than 80
  2. create another folder for proxy and put stuff like this in it 
                        
                    #reverse proxy
                    server {
                        listen 80;
                        server_name _;
                    
                        location / {
                            proxy_pass http://127.0.0.1:8000;
                        
                            proxy_set_header Host $host;
                            proxy_set_header X-Real-IP $remote_addr;
                            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                        }
                    }
  3. enable it and that's pretty much it

as you can guess, the additional lines are just ip packet type data

caching and rate limmiting

surprisingly very easy just add these to the top of proxy or load balancer

  • rate limitter: limit_req_zone $binary_remote_addr zone=mylimit:10m rate=5r/s;
  • caching: proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m max_size=1g inactive=60m;

then put these lines in whatever route you wanna apply this to

  • rate limitting: limit_req zone=mylimit burst=3 nodelay;
  • caching:proxy_cache my_cache; proxy_cache_valid 200 2s;

they are both referenced by the zone name and have parameters for where they are stored and requests per second/max size and inacetivity and bursts, which is a buffer btw

last but certainly not least, trying to do ssl/tls, well, kinda, gonna be local, in real life gotta hit the all cert

  • first gotta generate a signed certificat: sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout /etc/ssl/private/nginx-selfsigned.key -out /etc/ssl/certs/nginx-selfsigned.crt
  • add this overcomplicated script that basically has everything in it 
                 
             limit_req_zone $binary_remote_addr zone=gatewayssl_limit:10m rate=5r/s;
             
             # This is the line Nginx is complaining about missing:
             proxy_cache_path /var/cache/nginxssl levels=1:2 keys_zone=gatewayssl_cache:10m max_size=1g inactive=60m use_temp_path=off;
             
             
             server {
                 listen 443 ssl;
                 server_name _;
             
                 # SSL Files
                 ssl_certificate /etc/ssl/certs/nginx-selfsigned.crt;
                 ssl_certificate_key /etc/ssl/private/nginx-selfsigned.key;
             
                 # Helpful Header to see if Cache is working
                 add_header X-Cache-Status $upstream_cache_status;
             
                 location / {
                     # Assuming your site is in /var/www/crowsite
                     root /var/www/crowsite;
                     index index.html;
                     try_files $uri $uri/ =404;
                 }
             
                 # GATEWAY ROUTE: Users
                 location /users {
             
                     limit_req zone=gatewayssl_limit burst=10 nodelay; # Rate Limit
